У третій місії з серії про лампочки я хочу окремо виділити процес та період спостереження за цим процесом.

У попередній місії був введений параметр start_watching, а в цій - параметр end_watching, який передає час, коли необхідно закінчити спостереження. Якщо його не передано, місія працює як у попередній версії, без обмеження часу спостереження.

Ще одне оновлення полягає в тому, що кількість елементів (натискань кнопок) може бути непарною (раніше була умова, що кількість елементів завжди парна). У цьому випадку обов'язково присутні параметри початку і кінця перегляду.

Вхідні дані: Три аргументи і тільки перший необхідний, інші опціональні. Перший {% if is_js %}масив Date об'єктів{% else %}список datetime об'єктів{% endif %}, другий та третій {% if is_js %}Date об'єкт{% else %}datetime об'єкт{% endif %}.

Вихідні дані: Кількість секунд як ціле число.

Передумови:

  • {% if is_js %}Масив{% else %}Список{% endif %} натискання кнопки завжди сортується у порядку зростання.
  • {% if is_js %}Масив{% else %}Список{% endif %} натискання кнопки не має повторюваних елементів.
  • Мінімальна можлива дата - 01.01.1970
  • Максимальна можлива дата 31.12.9999
